# 情绪板方法论 — Mood Board Method

> 将模糊的"我想要好看"转化为可操作的视觉参数。
> 用于 Vision Step 2 的风格方向探索。

---

## 1. 情绪板结构（4 象限）

一个完整的情绪板覆盖 4 个视觉维度：

```
┌─────────────────────┬─────────────────────┐
│                     │                     │
│   Q1: Color +       │   Q2: Typography    │
│   Texture           │                     │
│                     │                     │
│   主色调、辅助色     │   字体选择、字号层级  │
│   渐变方向、纹理质感  │   字重组合、间距感受  │
│                     │                     │
├─────────────────────┼─────────────────────┤
│                     │                     │
│   Q3: Layout +      │   Q4: Motion +      │
│   Space             │   Interaction       │
│                     │                     │
│   网格策略、留白密度  │   入场方式、hover 反馈│
│   section 节奏      │   滚动行为、过渡风格  │
│                     │                     │
└─────────────────────┴─────────────────────┘
```

## 3. 参考提取协议

### 3.1 从参考中提取什么

| 维度 | 提取 | 具体化 |
|------|------|--------|
| 布局节奏 | 各 section 的高度比例和间距 | "Hero 100vh, Features auto, gap 80px" |
| 配色比例 | 主色/辅色/强调色的面积比 | "60% 深蓝 / 30% 白色文字 / 10% 蓝色 CTA" |
| 排版字阶 | 标题/正文/辅助的字号层级 | "Hero 72px, H2 36px, body 18px → 约 Perfect Fifth" |
| 动效语言 | 入场方式和滚动行为 | "stagger fade-up, 100ms delay per item" |
| 纹理质感 | 是否有渐变/grain/blur | "微妙 noise overlay + radial gradient bg" |
| 网格策略 | 列数和分割方式 | "12 列不对称 7:5, max-width 1200px" |

### 3.2 丢弃什么

- **品牌专属元素**：他人的 logo、品牌色、品牌字体
- **具体内容**：文案措辞、特定产品图片
- **技术实现细节**：框架选择、代码结构
- **装饰过度的元素**：不可复用的一次性装饰
- **违反 AG 红线的做法**：即使参考中有也不采用

### 3.3 提取模板

```markdown
## 参考分析：[URL/名称]

**提取**：
- 布局：[具体描述]
- 配色：[比例和色调]
- 排版：[字阶和处理方式]
- 动效：[语言和节奏]

**丢弃**：
- [品牌色 #xxx——替换为 AG 兼容色]
- [自定义字体——替换为系统字体或 TIER_3 display font]

**精华**：[一句话总结这个参考最值得借鉴的点]
```

## 4. 从广到窄的迭代协议

### Phase 1：广泛探索（10 → 5）
- 收集 10 个视觉参考（来自用户提供 + 风格原型推荐）
- 快速标注每个参考的"精华"（1 句话）
- 淘汰与 Brief 情绪不匹配的 5 个

### Phase 2：聚焦（5 → 3）
- 对剩余 5 个参考做详细提取（使用 §3 模板）
- 按 4 象限归类提取结果
- 合并为 3 个方向（Direction Card）

### Phase 3：定向（3 → 1）
- 呈现 3 个 Direction Card 给用户
- 用户选择 1 个（或混合）
- 进入 Vision Spec Step 3 深化

---

## 5. 无参考时怎么办

用户没有提供参考也不想找参考时：

1. 回到 §2 情绪关键词映射表
2. 取用户给出的 3-5 个情绪词
3. 在映射表中找交叉——哪些视觉属性值在所有关键词中一致？
4. 一致的属性 = 确定的方向；冲突的属性 = 需要用户选择
5. 从 style-vocabulary.md 中选匹配度最高的风格原型

```
示例：用户关键词 = "innovative" + "trustworthy" + "modern"

色温：冷（3/3 一致）→ 确定
饱和度：中（3/3 一致）→ 确定
对比度：高（3/3 一致）→ 确定
密度：中（2/3）vs 中-低（1/3）→ 偏中，确定
圆角：小-中（2/3）vs 中（1/3）→ 小-中，确定
动效：平稳-快速（有分歧）→ 提供两个方向让用户选

→ 匹配风格：Swiss / Glassmorphism / Scientific Precision（取决于动效选择）
```

## 6. AI 主动搜索设计灵感协议

> **核心原则**：不依赖用户提供参考——vision 自己去设计平台搜索高质量灵感。
> 用户不一定是设计师，可能不知道去哪里找参考、也不知道什么是"好的设计"。AI 必须主动承担这个工作。

### 6.1 搜索平台优先级

| 平台 | 强项 | 搜索方式 | 何时用 |
|------|------|---------|--------|
| **Dribbble** | UI/Web 设计、视觉系统 | 网页搜索 `site:dribbble.com [关键词]` | 首选——UI 设计质量最高 |
| **Behance** | 品牌/视觉/平面设计 | 网页搜索 `site:behance.net [关键词]` | 品牌页/视觉系统/完整案例 |
| **Awwwards** | 创意网站/交互 | 网页搜索 `site:awwwards.com [关键词]` | 寻找获奖级创意网站 |
| **Pinterest** | 情绪灵感/色彩/纹理 | 网页搜索 `site:pinterest.com [关键词]` | 情绪板素材/色彩灵感 |
| **Mobbin** | 移动端 UI 模式 | 网页搜索 `site:mobbin.com [关键词]` | 移动端页面参考 |

### 6.2 搜索关键词构造规则

```
基础公式：[行业/领域] + [页面类型] + [风格关键词] + [质量修饰词]

示例：
- "biotech landing page minimal design" → 生物科技 + Landing + 极简
- "SaaS dashboard dark mode premium" → SaaS + Dashboard + 暗色 + 高端
- "AI product hero section glassmorphism" → AI 产品 + Hero + 玻璃态
- "brand showcase website cinematic" → 品牌展示 + 网站 + 电影感

质量修饰词（提高搜索结果质量）：
- "best" / "award" / "premium" / "top" / "featured"
- "2024" / "2025"（时效性）
- "web design" / "UI design"（聚焦领域）
```

### 6.3 筛选标准

从搜索结果中筛选高质量参考，不是随便搜到什么都用：

| 维度 | 标准 | 排除 |
|------|------|------|
| **质量** | Dribbble ≥500 likes / Behance Featured | <100 likes 的练习稿 |
| **来源** | 认证设计师、知名工作室（Ramotion/Tubik/Fantasy 等） | 匿名/初学者作品 |
| **完成度** | 视觉完成度高的完整作品 | 草图/线框图/概念稿 |
| **可执行性** | Web 技术可实现（HTML/CSS/JS） | 重度依赖视频/3D/WebGL |
| **时效性** | 2 年内的作品优先 | 过度老旧的设计趋势 |
| **相关性** | 行业/页面类型与 Brief 匹配 | 完全不相关的领域 |

### 6.4 呈现与提取

1. 筛选后保留 **3-5 个** 高质量参考（不多不少）
2. 对每个参考用 §3 的提取模板进行结构化提取：

```markdown
## 参考分析：[作品名 / 设计师]

**来源**：[平台 + URL]
**行业**：[作品所属行业]

**提取**：
- 布局：[具体描述——如"Hero 100vh + 居中标题 + 底部产品截图"]
- 配色：[比例和色调——如"60% 深蓝黑 / 30% 灰白文字 / 10% 紫色渐变"]
- 排版：[字阶——如"Hero 72px, H2 36px, body 16px → Perfect Fourth"]
- 动效：[语言——如"stagger fade-up, 100ms delay"]
- 纹理：[描述——如"微妙 noise overlay + radial gradient glow"]

**丢弃**：
- [品牌 logo/色 → 替换为 AG 兼容]
- [特定字体 → 替换为系统字体或 TIER_3 display font]

**精华**：[一句话——这个参考最值得借鉴的核心亮点]
```

3. 综合 3-5 个参考的提取结果，合并为 3 个 Direction Card（见 SKILL.md Step 2.3）

### 6.5 搜索无结果的回退

```
设计平台搜索失败？
│
├─ 关键词太窄 → 放宽关键词（去掉行业限定，只保留风格词）
├─ 网络不可达 → 回退到 style-vocabulary.md 原型匹配
├─ 结果质量不足 → 用 §2 情绪关键词映射表 + §5 无参考时的关键词交叉法
└─ 向用户说明："未找到足够高质量的参考，以下方向基于内置风格原型库推荐"
```
