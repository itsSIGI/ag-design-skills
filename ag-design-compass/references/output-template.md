# 输出模板

> Design Compass 各 Step 的标准输出格式。

---

## 角色卡片模板（Step 3）

```markdown
### 角色：[角色名称]

**角色背景**
- 工作场景：[实验室 / 办公室 / 远程]
- 使用设备：[台式机 / 笔记本 / 平板]
- 技术熟练度：[高 / 中 / 低]

**当前行为**
- [描述该角色目前实际如何完成任务]
- 来源：[引用原文段落]

**核心诉求**
> "我希望能 [一句话描述]"

**当前痛点**
1. [痛点 1] — 来源：[原文引用] [待验证]?
2. [痛点 2] — 来源：[原文引用]
```

---

## 操作动线模板（Step 3）

```markdown
### 操作动线

[角色 A] → 触发事件 → [角色 B 响应] → [系统处理] → [角色 C 查看结果]

详细流程：
1. [角色 A] 在 [场景] 中 [操作]
2. 系统 [自动/手动] 处理
3. [角色 B] 收到 [通知/结果]，进行 [操作]
4. ...
```

---

## 设计目标模板（Step 4）

```markdown
### 需求目标
1. 解决 [角色] 在 [场景] 中 [痛点]
2. ...

### 设计目标
让 [角色] 在 [关键场景] 中能够 [获得/完成什么体验结果]，同时 [满足一个关键约束]

### 自检结果
| 原则 | 通过? | 说明 |
|------|-------|------|
| 可衡量 | ✅/❌ | [指标] |
| 合理性 | ✅/❌ | [说明] |
| 可执行 | ✅/❌ | [说明] |
| 时效性 | ✅/❌ | [说明] |
```

---

## 设计策略模板（Step 5）

```markdown
### 策略 1：[机制级策略名称]

- **策略描述**：[系统如何运作 / 规则如何建立]
- **解决的问题**：角色 [X] 的痛点 #[N]
- **关键因素**：[决定效果的变量]
- **需验证的假设**：[执行前需确认的前提]
- **优先级**：MVP / 后续迭代
- **使用框架**：[FOGG / JTBD / ...]，因为 [选择原因]
```

---

## UI 模块规格表模板（Step 5.5）

```markdown
### 模块：[模块名称]

| 字段 | 值 |
|------|-----|
| 平台 | Web / Mobile / Both |
| 使用角色 | [角色名] |
| 优先级 | MVP / 后续迭代 |
| 推荐页面原型 | [archetype 名称] |
| 推荐框架 | React / Vue |

**信息字段**（已过剃刀）：
- [字段 1]：[类型] — [必要性理由]
- [字段 2]：[类型] — [必要性理由]

**交互元素**：
- [按钮/输入/选择器]：触发 [行为]

**状态变体**：
- 默认态：[描述]
- 空态：[描述]
- 错误态：[描述]

→ 交给 ag-design-craft
```
