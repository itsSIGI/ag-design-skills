# 业务上下文索引

> Design Compass 不内置业务上下文。所有业务知识由团队文档统一管理。
> 本文件是索引——告诉 AI 去哪里找信息，而不是复制信息。

---

## 核心业务领域

### 1. 化学数据库管理
- **做什么**：化合物数据的存储、检索、管理、共享
- **关键实体**：化合物（Compound）、数据集（Dataset）、分子属性（Property）
- **典型角色**：研究员、数据管理员
- **信息来源**：产品需求文档、用户访谈

### 2. 实验流程管理
- **做什么**：实验设计、执行、记录、分析的全流程管理
- **关键实体**：实验（Experiment）、步骤（Step）、参数（Parameter）、结果（Result）
- **典型角色**：研究员、实验员、PI
- **信息来源**：实验室 SOP、用户调研

### 3. AI 模型 / 计算任务
- **做什么**：计算化学任务提交、监控、结果分析；AI 模型训练与部署
- **关键实体**：计算任务（Task）、模型（Model）、资源（Resource）
- **典型角色**：计算化学家、数据科学家、平台开发者
- **信息来源**：技术文档、HPC 管理文档

### 4. AI Agent / 对话系统
- **做什么**：AI Agent 配置、对话交互、工具调用编排
- **关键实体**：Agent、对话（Conversation）、工具（Tool）、提示词（Prompt）
- **典型角色**：Agent 开发者、终端用户
- **信息来源**：Agent SDK 文档、用户反馈

### 5. 协作与管理
- **做什么**：团队协作、项目管理、权限控制、审计追踪
- **关键实体**：项目（Project）、团队（Team）、成员（Member）、权限（Permission）
- **典型角色**：PI、项目经理、管理员
- **信息来源**：组织结构文档

## 使用方式

当 Step 3-5 缺业务背景时：
1. 先查本索引，定位到哪个领域
2. 向用户索要该领域的具体文档（需求文档、用户访谈记录、竞品分析等）
3. 不要用本索引的简要描述替代真实业务文档——这里只是地图，不是地形
