# 示例：生物科技公司品牌官网 Landing Page

> 本示例覆盖 compass Step 1 → Step 5.5 全流程，包含 Step 5 策略拆解和视觉创新标记。
> 展示如何将品牌展示型需求拆解为可执行的 UI 模块规格，并触发 vision skill。

## 背景

BioGenesis（虚构）是一家 AI 驱动的药物发现公司，正在筹备品牌官网上线。
核心需求："做一个展示我们技术实力和平台能力的 Landing page，面向投资人和潜在客户。要有高级感。"

## Step 2: 文档摄入

**文档类型**：功能简报（用户口述）

**提取的结构化信息**：

| 维度 | 内容 | 来源 |
|------|------|------|
| 业务目标 | 展示 AI 药物发现技术实力，吸引 B 轮投资人和大药厂合作 | 用户口述 |
| 目标受众 | 投资人（VC/PE）、大型药企 BD 负责人 | 用户口述 |
| 核心信息 | 1) AI 平台能力 2) 管线进展 3) 团队背景 | 用户口述 |
| 情绪关键词 | "高级感"、"可信赖"、"创新" | 用户口述 |
| 已有约束 | 有 logo 但无完整品牌指南；技术栈 React + Next.js | 用户口述 |
| 已知痛点 | "现在没有官网，投资人只能看 PPT" | 用户口述 |

**缺口处理**：
> "目前缺少品牌指南文档。是否有 logo 文件、品牌色定义、字体规范？如果没有，后续视觉方向会走全新配色模式。"

用户回复：只有 logo（深蓝色），没有完整品牌指南。

## Step 3: 角色识别与操作动线

### 角色 1：VC 投资人

- **角色背景**：管理 5-10 亿基金，每周看 10+ 个 biotech 项目。桌面端为主，时间极有限，30 秒内判断是否值得深入。
- **当前行为**：收到 BP 后打开附件 PPT，快速翻看团队/管线/数据页。
- **核心诉求**："我希望 30 秒内看到这家公司的核心数据和差异化。"
- **当前痛点**：
  - "没有官网，看起来不太正规"——来源：用户转述投资人反馈
  - "PPT 信息密度太高，找不到重点"——来源：用户转述 `[待验证 — 仅 1 个来源]`

### 角色 2：大药企 BD 负责人

- **角色背景**：负责外部合作评估，需要评估技术平台能力和管线匹配度。
- **当前行为**：通过行业会议和推荐了解公司，然后要求发详细资料。
- **核心诉求**："我需要快速判断他们的 AI 平台和我们的管线方向是否匹配。"
- **当前痛点**：
  - "对方技术能力描述太抽象，缺少具体数据"——来源：用户转述 BD 反馈 `[待验证 — 仅 1 个来源]`

### 操作动线

```
投资人/BD → Google 搜索/推荐链接 → Landing Page
  → Hero（30s 判断）
  → 技术平台介绍（1-2min 了解核心能力）
  → 管线进展（关键数据）
  → 团队背景（信任建立）
  → CTA（联系/下载 BP）
```

## Step 4: 需求目标 → 设计目标

### 业务诉求分析

| 维度 | 内容 |
|------|------|
| 目标用户 | VC 投资人、大药企 BD |
| 核心价值 | 展示 AI 药物发现技术实力和管线进展 |
| 价值表现 | 专业、可信赖、创新的品牌形象 |
| 实现策略 | 品牌官网 Landing page |

### 用户诉求分析

> 在评估 biotech 公司的场景下，投资人/BD 快速浏览官网来判断公司技术实力和合作价值。

### 产出

**需求目标**：解决投资人和 BD 在评估公司时"缺少专业线上展示渠道、关键信息分散"的问题。

**设计目标**：让投资人/BD 在首次访问 Landing page 的 60 秒内能够清晰理解公司的 AI 平台核心能力和管线进展，同时建立"专业可信赖"的品牌认知。

### 自检

- [x] 可衡量：60 秒内理解核心能力（可通过用户测试验证）
- [x] 合理性：Landing page 可实现
- [x] 可执行：React + Next.js 技术栈匹配
- [x] 不是把行动当目标：目标是"理解 + 品牌认知"，不是"做一个页面"

## Step 5: 设计策略拆解

### 5.1 发散切入点

| 框架 | 候选方向 |
|------|---------|
| **AIDA** | Attention（Hero 震撼）→ Interest（平台能力）→ Desire（数据证明）→ Action（CTA） |
| **用户旅程** | 首次触达 → 快速扫描 → 深度阅读 → 行动转化 |
| **FOGG** | 动机（看到价值）× 能力（信息易获取）× 触发（CTA 明显） |

### 5.2 归纳排序

收敛为 3 条策略，按契合度 × 影响力排序。

### 5.3 策略详细化

#### 策略 1：渐进式信息披露机制

- **策略描述**：按投资人决策流程编排信息层级——30 秒层（Hero 核心数据）、2 分钟层（平台能力详解）、5 分钟层（管线数据+团队），每层有独立 CTA 出口。
- **解决的问题**：投资人（角色 1）"PPT 信息密度太高，找不到重点"
- **关键因素**：Hero 区核心数据选择、各层信息密度控制
- **需验证的假设**：投资人是否会按顺序滚动浏览（vs 直接跳到特定区域）
- **优先级**：MVP
- **框架**：AIDA — 因为受众是首次访问者，符合注意力→兴趣→欲望→行动的线性路径

#### 策略 2：数据可视化信任建立机制

- **策略描述**：用结构化的 KPI 数据（管线数量、化合物库规模、AI 预测准确率等）替代抽象的文字描述，配合动态计数和简洁图表增强说服力。
- **解决的问题**：BD（角色 2）"技术能力描述太抽象，缺少具体数据" `[待验证]`
- **关键因素**：数据真实性、数据选择（哪些指标最能打动投资人）
- **需验证的假设**：KPI 数字是否已准备好（用户需提供）
- **优先级**：MVP
- **框架**：FOGG 能力维度 — 降低理解成本，用数字替代文字

#### 策略 3：品牌信任符号嵌入机制

- **策略描述**：在关键决策节点嵌入信任符号（合作伙伴 logo、媒体报道、团队学术背景、专利数据），不作为独立板块而是与内容融合。
- **解决的问题**：投资人和 BD 对早期公司的天然不信任
- **关键因素**：信任符号的真实性和放置位置
- **需验证的假设**：公司是否有足够的信任符号可展示
- **优先级**：MVP（基础信任符号）+ 后续迭代（深度案例）
- **框架**：用户旅程 — 在"犹豫"触点植入信任信号

### 5.4 自检

- [x] 系统性：3 条策略覆盖信息结构（策略 1）→ 内容说服力（策略 2）→ 信任建立（策略 3），全局关联
- [x] 多样性：分别来自 AIDA、FOGG、用户旅程 3 个框架
- [x] 精准性：每条策略对应具体角色痛点

**待验证假设清单**：
1. KPI 数据是否已准备好
2. 有哪些合作伙伴/媒体报道可展示
3. 投资人是否按顺序浏览（需用户测试验证）

## Step 5.5: UI 模块规格表

| 模块 | 平台 | 角色 | 优先级 | 信息字段 | 交互元素 | 推荐原型 | 视觉创新 |
|------|------|------|--------|---------|---------|---------|---------|
| Hero 品牌区 | Web | 投资人、BD | MVP | 品牌标语、1-2 句核心价值主张、1 个核心 KPI | 主 CTA（联系/预约 Demo）、次 CTA（了解更多） | — | **是** |
| AI 平台能力展示 | Web | BD | MVP | 平台 3 大核心能力、每个能力配图/图标+简述 | 可选 tab 切换或滚动 | — | **是** |
| 管线数据面板 | Web | 投资人 | MVP | 管线数量、阶段分布、核心管线名称+阶段 | 无（只读） | — | **是** |
| KPI 数据带 | Web | 投资人、BD | MVP | 化合物库规模、AI 预测准确率、合作客户数 | 数字计数动画（滚动触发） | — | **是** |
| 合作伙伴/信任符号 | Web | 投资人、BD | MVP | 合作伙伴 logo 列表、媒体报道引用 | 无 | — | **是** |
| 团队背景 | Web | 投资人 | MVP | 核心团队成员头像+title+学术背景 | 无 | — | **是** |
| CTA / 联系区 | Web | 投资人、BD | MVP | 联系方式、预约表单 | 表单提交、日历预约 | — | **是** |
| 导航栏 | Web | 全部 | MVP | Logo、页内锚点链接 | 锚点跳转、CTA 按钮 | — | 否 |
| 页脚 | Web | 全部 | MVP | 版权、社交链接、法律链接 | 链接跳转 | — | 否 |

**路由说明**：

- 视觉创新 = **是**（Hero ~ CTA 区）→ **先交给 ag-design-vision 产出 Vision Spec → 再交给 ag-design-craft**
  - 配色模式建议：全新配色（无完整品牌指南）
  - 情绪关键词传递：高级感、可信赖、创新
- 视觉创新 = 否（导航栏、页脚）→ **直接交给 ag-design-craft**（走 AG 标准 TIER_1 规范）
