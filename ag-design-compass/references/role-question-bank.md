# 角色追问题库 — 化学科研 & AI 平台领域

> 当 Step 3 角色信息不足时，按角色类型选择对应追问模板。
> 追问目的：补全角色卡片中"当前行为"和"核心痛点"的信息缺口。

---

## 研究员 / 化学家

| 维度 | 追问 |
|------|------|
| 工作场景 | 这位研究员主要在实验室还是办公室工作？日常用什么设备（台式机/笔记本/平板）？ |
| 当前行为 | 他们现在怎么查找化合物数据？用什么工具管理实验记录？手动还是自动化？ |
| 数据管理 | 实验数据存在哪里？Excel / 内部系统 / 纸质记录？跨团队如何共享？ |
| 计算需求 | 需要跑什么类型的计算？DFT / MD / ML？等待结果的时间有多长？ |
| 痛点方向 | 最花时间的重复性工作是什么？数据从实验到分析的流转有哪些断点？ |
| 协作 | 需要和哪些角色协作（PI/实验员/计算化学家）？协作中最大的摩擦是什么？ |

## 实验员 / 实验技术人员

| 维度 | 追问 |
|------|------|
| 工作场景 | 主要在湿实验室还是干实验室？操作时能看屏幕吗？手是否经常被占用？ |
| 当前行为 | 实验步骤怎么记录？实时记录还是事后补录？用什么工具？ |
| 数据采集 | 仪器数据怎么导出？自动采集还是手动读数？ |
| 痛点方向 | 记录实验参数时最容易出错的环节？重复操作最多的步骤？ |
| 安全合规 | 有没有实验安全检查清单？化学品管理怎么做？ |

## 计算化学家 / 数据科学家

| 维度 | 追问 |
|------|------|
| 工作场景 | 主要用什么计算工具（Gaussian/ORCA/VASP/自研脚本）？在本地还是集群上跑？ |
| 当前行为 | 怎么提交计算任务？命令行 / Web UI / 脚本？结果怎么可视化？ |
| 数据管理 | 计算结果存在哪里？怎么和实验数据关联？ |
| 痛点方向 | 排队等资源的时间有多长？任务失败后的调试流程是什么？ |
| AI/ML | 有没有用机器学习做预测？训练数据怎么来？模型怎么评估？ |

## PI / 项目负责人

| 维度 | 追问 |
|------|------|
| 管理范围 | 管几个课题/项目？团队多大？跨地域吗？ |
| 当前行为 | 怎么了解项目进度？多久开一次组会？ |
| 决策场景 | 什么时候需要看数据做决策？需要什么粒度的数据（汇总/明细）？ |
| 痛点方向 | 信息获取的最大瓶颈是什么？有没有因为信息滞后导致的决策失误？ |
| 合规/发表 | 数据可追溯性要求高吗？对实验记录完整性有什么规定？ |

## Agent 开发者 / 平台开发者

| 维度 | 追问 |
|------|------|
| 工作场景 | 开发什么类型的 Agent？对接哪些工具/API？ |
| 当前行为 | 怎么测试 Agent？有没有评估框架？ |
| 调试 | Agent 出错时怎么排查？日志够用吗？ |
| 痛点方向 | 最耗时的开发环节是什么？最常见的 Agent 失败模式？ |
| 部署 | 怎么部署和监控 Agent？有没有灰度/回滚机制？ |

## 通用追问（所有角色适用）

| 维度 | 追问 |
|------|------|
| 频率 | 这个操作多久做一次？每天/每周/每月？ |
| 紧急程度 | 这个任务有时间压力吗？延迟的后果是什么？ |
| 替代方案 | 如果这个功能不存在，他们怎么解决这个问题？ |
| 成功标准 | 他们认为"做好了"的标准是什么？ |
| 情绪 | 做这件事时的情绪状态？焦虑/无聊/紧张/自信？ |
