# Multi-Page Flow — 多页面流程协议

> 适用于向导（wizard）、onboarding 序列、多步表单、跨页面操作流等需要**多个页面保持一致性**的场景。

## 触发条件

用户需求涉及 ≥2 个页面且存在以下任一：
- 页面间有**导航顺序**（上一步/下一步）
- 页面间有**共享状态**（步骤 1 填的数据在步骤 3 展示）
- 页面间有**视觉连贯性**要求（同一业务流的不同阶段）

## 执行流程

### F1：流程总览（在 Step 1 之前）

先输出**流程地图**，锁定整体结构：

```markdown
## Flow Map

| 序号 | 页面 | 页面原型 | 核心任务 | 共享数据 | 入口条件 |
|------|------|---------|---------|---------|---------|
| 1 | 基础信息 | create-edit-form | 填写项目名称+类型 | projectName, projectType | 直接进入 |
| 2 | 参数配置 | create-edit-form | 配置计算参数 | params{} | 步骤 1 完成 |
| 3 | 确认提交 | entity-detail | 预览+提交 | 全部 | 步骤 2 完成 |
| 4 | 结果展示 | dashboard | 查看结果 | resultId | 提交成功 |
```

### F2：共享 Recipe 层

多页面流程的 Recipe 分两层：

**Flow-level Recipe（共享层）**——所有页面共用：
- 导航组件（步骤条 / 面包屑 / 进度指示器）
- 页面容器和全局布局
- 共享的状态管理方案
- 公共按钮组（上一步/下一步/取消）

**Page-level Recipe（页面层）**——每页独立：
- 各页面自己的 slot 和组件
- 页面内的局部状态

### F3：一致性检查清单

每个页面生成后，过以下跨页面检查：

- [ ] **导航一致**：步骤条样式、位置、交互在所有页面相同
- [ ] **按钮位置一致**：主操作按钮（下一步/提交）在所有页面同一位置
- [ ] **间距节奏一致**：页面 padding、section gap 跨页面相同
- [ ] **容器层级一致**：同层级内容在不同页面用同种 `<AgCard>` 变体
- [ ] **状态传递完整**：Flow Map 中的共享数据在对应页面可访问
- [ ] **空态/错误态**：每个页面都处理了前置步骤未完成的边界情况
- [ ] **回退路径**：用户能从任意步骤回到上一步，且填写内容不丢失

### F4：输出方式

**逐页生成，但始终可见 Flow Map**：
1. 输出 Flow Map + Flow-level Recipe → 用户确认
2. 逐页按标准 Step 1-4 流程生成，每页 Page-level Recipe 独立确认
3. 全部页面完成后，跑一次 F3 跨页面一致性检查
4. Decision Summary 中增加 `Flow: [流程名] Page [N/M]` 标注

## 页面原型推荐

| 流程类型 | 推荐组合 |
|---------|---------|
| 创建向导 | create-edit-form × N + entity-detail（确认页） |
| Onboarding | create-edit-form + dashboard（完成后首页） |
| 数据导入 | create-edit-form（配置）+ computation-dashboard（进度）+ entity-detail（结果） |
| 审批流 | list-table（待审列表）+ entity-detail（审批详情） |
