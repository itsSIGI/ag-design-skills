# 设计宪法 — Agentic Genius

本文件沉淀 AG 设计决策的裁决规则，供 Agent 在做方案比较、审查和验收时按需加载。

---

## 1. 核心公理与优先级

按以下顺序裁决，**不可倒序**：

1. **Trust（信任度）**
   任何隐藏信息、诱导点击、默认转嫁代价、隐私侵犯都直接 `REJECT`。
   化学科研场景中，数据精度虚标、AI 结果置信度不透明等同于欺骗。

2. **Task Efficacy（任务效能）**
   优先选择认知负荷更低、步骤更短、主次关系更清晰的方案。
   科研工具的核心用户是研究员——他们的时间极其昂贵，每多一步操作都是成本。

3. **Accessibility（无障碍）**
   不满足 WCAG 2.2 AA 对比度（4.5:1 正文 / 3:1 大文本）或核心交互键盘不可达时直接 `REJECT`。

## 2. 一票否决红线

命中任一条即 `FATAL_ERROR`：

### 2.1 欺骗性设计模式（Dark Pattern）
- 弱化取消、退订、返回等反向操作到不可见或近乎不可见
- 伪造倒计时、伪造紧张感、默认勾选带隐藏费用的选项
- AI 生成的结果未标注来源或置信度

### 2.2 体验死胡同
- 用户到达关键页面后没有明确下一步
- 提交操作后没有任何可点击按钮或明确去向
- 返回操作直接清空未保存数据且无拦截提示

### 2.3 无障碍崩坏
- 正文文本与背景对比度 < 4.5:1
- 大号文本或非文本交互元素对比度 < 3:1
- 核心点击热区小于 44×44 CSS px
- 仅靠颜色表达错误、成功、警告等关键状态
- 键盘无法到达核心交互，或焦点不可见
- 用户启用减少动效后仍强制播放明显位移/缩放动画

### 2.4 品牌污染
- Logo 非等比缩放、拉伸或随意变色
- 品牌蓝/绿出现在非指定的三处（链接 hover / agent 状态点 / 代码高亮）
- 焦点环或选中态使用品牌蓝（应为中性 `--color-focus-ring`）
- 按钮填充使用彩色（Primary 应为黑/白中性色）
- 字重使用 700（上限为 600）

### 2.5 工程化脱轨
- 业务代码中出现裸十六进制色值（无 `TOKEN_ESCAPE` 声明）
- 手写 `:focus` / `:hover` 规则覆盖 shadcn/AG 组件内置交互态
- 使用非 MingCute 图标库
- 引入 Web Font / Google Fonts
- 自定义圆角或阴影值（不从 token 取值）

### 2.6 化学科研专项红线
- 数据精度虚标：显示精度超过实际测量精度
- 不可追溯：实验数据修改无审计日志
- AI 可信度模糊：模型预测结果不展示置信区间或数据来源
- 分子结构渲染失真：2D/3D 渲染与实际构型不一致时未加警告
- 单位混淆：同一页面内混用不同单位制且无转换标注

## 3. 补充原则

### 3.1 适用范围
- 默认验收范围是 Light Theme 主链路
- 暗色模式通过 `[data-theme="dark"]` + 语义 token 自动适配，组件代码不应有任何改动

### 3.2 语义优先
- 先定义组件承担的语义角色，再谈具体颜色或尺寸
- 不要把单一色值或数值当作设计意图本身

### 3.3 状态等价
- 桌面鼠标、触屏、键盘可达场景要保持等价结果、等价反馈、等价风险保障
- 设备能力不同可以改变呈现形式，但不能改变结果和风险等级

### 3.4 跨场景一致
- 同一语义在不同页面和流程中保持同等视觉与交互含义
- 禁止"同语义多套表达"或"同表达多重语义"
- agent 状态点颜色严格绑定：绿=running/completed、蓝=idle、红=error、灰=paused

### 3.5 复杂度预算
- 每个界面必须有明确主任务和次级任务
- 任何新增元素若不能提升完成率、理解效率或风险可控性，默认不引入
- 每个任务区块最多一个主操作；次级操作必须明显降权

### 3.6 可访问的语义表达
- 状态信息至少通过颜色 + 文案/图形/结构中的一种组合表达
- 不能只把颜色变红就算错误提示完成

### 3.7 空间弹性
- 不要用固定宽高锁死文本容器
- 优先用 padding、最小尺寸、flex 或 grid 撑开容器
- 组件必须能吸收至少 50% 的文案增长

### 3.8 性能与动效预算
- 微交互动效不超过 200ms
- Collapse / Drawer / Modal 等空间重排可适度延长至 250ms
- 动效只服务反馈和空间隐喻，不得阻断任务流

### 3.9 AG 视觉签名
- **冷峻精确**：页面底纯白 `--color-bg`，卡片用 `<AgCard>`，无多余装饰
- **克制信号**：品牌色极少使用，大面积靠中性色层级传递信息
- **开发者优先**：信息密度适中偏高，减少"大留白"，代码/数据优先级高于装饰

## 4. 规则引擎

### 4.1 Purposeful（有目的）
- 装饰图形、复杂动效、额外文案必须服务任务或降低理解成本
- 否则移除（奥卡姆剃刀）

### 4.2 Restrained And Focused（克制聚焦）
- 每个任务区块最多一个主按钮
- 次级操作必须弱于主操作
- 品牌色只在指定三处出现

### 4.3 Consistent（一致）
- 返回、取消、确认、错误提示等基础交互继承全局模式
- 不要自行发明新的基础交互模式

### 4.4 Forgiving（容错）
- 删除、危险操作必须匹配确认或撤销机制
- 异常、空状态、断网必须提供解释和下一步按钮
- 不要把系统原始报错直接暴露给用户

### 4.5 Tokenized Engineering（Token 化工程）
- 颜色、间距、圆角、字体、阴影 100% 映射到 token
- 极端场景允许豁免，但必须写 `TOKEN_ESCAPE: [具体原因]`

### 4.6 Empathetic UX Writing（共情文案）
- 错误文案遵循：发生了什么 + 为什么（可选）+ 如何解决
- 语气客观、赋能，不推责、不指责用户
- 禁止"非法"、"禁止"、"你必须"等用词

## 5. 验收协议

### 5.1 五维验收

| 维度 | 检查项 |
|------|--------|
| **Trust** | 关键信息完整可见？用户能轻松执行反向选择？AI 结果有置信度？ |
| **Task Efficacy** | 主路径足够短？次级任务不干扰主任务？ |
| **Accessibility** | 对比度达标？状态不依赖单一颜色？键盘可达？ |
| **Consistency** | 同语义跨页面一致？token 使用规范？ |
| **Recovery** | 误操作有恢复路径？危险操作有确认？ |

### 5.2 AG 专项验收

| 检查项 | 标准 |
|--------|------|
| 品牌色使用 | 蓝/绿只在三处 |
| 焦点/选中态 | 中性色 `--color-focus-ring` / `--color-selected` |
| 按钮颜色 | Primary=黑白中性，Danger=红，其余中性 |
| 字重 | ≤ 600 |
| 图标 | MingCute line 默认 / fill 选中 |
| 圆角 | 按钮/输入 6px / 卡片 8px / 弹窗 12px / tag 4px |
| 阴影 | 从 `--shadow-*` 取，暗色下降级为边框 |

### 5.3 判定输出

- 命中红线：`REJECT`
- 五维 + AG 专项全部通过：`PASS`
- 无红线但有可修复缺陷：`CONDITIONAL_PASS`（附整改清单）
- `CONDITIONAL_PASS` 不可直接发布，必须整改后重验

## 6. 架构边界

| 层 | 说明 | 可操作性 |
|----|------|---------|
| **TIER_1 绝对统一** | tokens.css 定义的所有语义变量、shadcn/AG 组件、6 条铁律 | 只读，不可创新 |
| **TIER_2 业务定制** | 特定业务主题色（从装饰色板取）、业务插画、复合数据展示 | 可在 TIER_1 基础上组合 |
| **TIER_3 局部创新** | 实验性页面、运营活动 | 可创新，但不能替换主干链路 UI |
