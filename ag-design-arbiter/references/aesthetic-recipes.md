# 审美食谱 — Agentic Genius

> 非设计师最需要的"该选什么"决策树。
> 当你面对视觉选择犹豫不决时，按本文件的配方走，保证"不会出错"。
> 这不是创意指南，是"安全下限"——按配方做，视觉质量不会低于 70 分。

---

## 1. 配色选择树

### 1.1 页面级配色方案

```
你的页面是什么类型？
│
├─ 数据密集（表格/仪表板/数据浏览器）
│  → 白底 --color-bg + 卡片 --color-surface
│  → 文本只用 primary / secondary / muted 三级
│  → 强调数据用字号差异，不用颜色
│
├─ 表单/配置
│  → 白底 --color-bg
│  → 输入区域 --color-surface
│  → 分组用 subtle 分割线，不用色块分区
│
├─ 对话/交互
│  → 白底 --color-bg
│  → 用户气泡 --color-surface，Agent 气泡 --color-bg + border
│  → Agent 状态点是唯一彩色元素
│
└─ 详情/报告
   → 白底 --color-bg
   → 内容卡片 <AgCard>
   → 图表颜色从装饰色板按顺序取（见 §1.4）
```

### 1.2 状态色使用规则

```
你要表达什么状态？
│
├─ 成功/完成 → 绿色 --color-status-success-*
├─ 警告/注意 → 黄色 --color-status-warning-*
├─ 错误/失败 → 红色 --color-status-error-*
├─ 信息/提示 → 蓝色 --color-status-info-*
├─ 默认/中性 → 灰色 --color-status-default-*
│
└─ 分类/标签（非状态语义）
   → 从装饰色板取，不与状态色重叠
   → 模式：<color>100 做背景 + <color>800 做文字
```

### 1.3 "我该用什么颜色"速查

| 场景 | 答案 |
|------|------|
| 主文本 | `--color-text-primary`（黑/白） |
| 辅助文本 | `--color-text-secondary`（中灰） |
| placeholder | `--color-text-muted`（浅灰） |
| 按钮 | 黑白中性（Primary）/ 灰（Secondary）/ 红（Danger） |
| 链接 | 正常态用 `--color-text-primary`，hover 态用 `--color-link-hover`（唯一品牌蓝） |
| 焦点环 | `--color-focus-ring`（中性黑/白） |
| 卡片背景 | `--color-surface`（微灰） |
| 页面背景 | `--color-bg`（纯白） |
| 分割线 | `--color-border-subtle` |

### 1.4 图表/多系列配色方案

从装饰色板按以下顺序取色，确保视觉区分度：

```
序列 1: Teal     --t500
序列 2: Purple   --p-purple-500
序列 3: Orange   --o500
序列 4: Sky      --s500
序列 5: Pink     --p-pink-500
序列 6+: 从 DESIGN.md §2.6 完整色板继续取
```

规则：
- 不与状态色（红/绿/黄/蓝）重叠
- 同一图表最多 6 个系列，超过则合并为"其他"
- 图例文字用 `--color-text-secondary`

## 2. 字号组合配方

### 2.1 标准页面配方

```
页面标题    <PageTitle>       28px / 600
区块标题    <Heading>          22px / 600
子标题      <Subheading>       18px / 500
正文        <Body>             15px / 400
辅助文字    <Small>            13px / 400
说明/时间   <Caption>          12px / 400
```

**黄金法则**：相邻两级字号差 ≥ 2px，且不超过一倍。

### 2.2 数据密集页面配方

数据页需要更紧凑的层级：

```
页面标题    <Heading>          22px / 600
列标题      <Small>            13px / 500（medium）
数据值      <Body>             15px / 400
辅助数据    <Caption>          12px / 400
```

### 2.3 仪表板 KPI 配方

```
数字        <Display>          36px / 600
标签        <Small>            13px / 400
趋势/变化   <Caption>          12px / 500 + 状态色
```

### 2.4 对话界面配方

```
消息正文    <Body>             15px / 400
时间戳      <Caption>          12px / 400
发送者名    <Small>            13px / 500
系统提示    <Small>            13px / 400（muted 色）
代码块      <Body>             15px / 400（mono 字体）
```

## 3. 间距节奏配方

### 3.1 页面级节奏

```
页面顶部 padding      --space-8   (32px)
区块间距              --space-8   (32px)
卡片内 padding        --space-6   (24px)
卡片间距              --space-4   (16px)
表单字段间距          --space-4   (16px)
紧凑列表项间距        --space-2   (8px)
标题与内容间距        --space-3   (12px)
图标与文字间距        --space-icon-text (6px)
```

### 3.2 "我该用多大间距"速查

```
两个元素之间的关系是？
│
├─ 同一组件内部（图标+文字、标签+值）
│  → --space-1 (4px) 或 --space-icon-text (6px)
│
├─ 同一区块内的同级元素（列表项之间、表单字段之间）
│  → --space-3 (12px) 或 --space-4 (16px)
│
├─ 不同区块之间（标题与内容、卡片与卡片）
│  → --space-4 (16px) 或 --space-6 (24px)
│
└─ 大区域分隔（页面区块之间）
   → --space-8 (32px) 或 --space-12 (48px)
```

### 3.3 呼吸感法则

- 内容越重要，周围留白越多
- 信息密度高的区域（表格/代码）允许更紧凑
- 但即使最紧凑的区域，最小间距也是 `--space-1`（4px）

## 4. 布局预设模式

### 4.1 侧边栏 + 主内容

```
┌─ Sidebar (240-280px fixed) ─┬─ Main Content (flex: 1) ──────┐
│                              │                                │
│  导航项 padding: --space-3   │  页面顶 padding: --space-8     │
│  导航项间距: --space-1       │  内容 max-width: 1200px        │
│  分组间距: --space-4         │  卡片用 <AgCard>              │
│                              │                                │
└──────────────────────────────┴────────────────────────────────┘
```

适用于：仪表板、数据浏览器、设置页、Agent 配置

### 4.2 全宽表格

```
┌─ 工具栏（搜索 + 筛选 + 操作）────────────────────────────────┐
│  padding: --space-4                                           │
├───────────────────────────────────────────────────────────────┤
│  表格（表头 --space-3 padding，行 --space-2 padding）          │
│  行 hover: --color-surface-hover                              │
│  选中行: --color-selected 左边框                               │
├───────────────────────────────────────────────────────────────┤
│  分页栏 padding: --space-3                                    │
└───────────────────────────────────────────────────────────────┘
```

适用于：化合物列表、实验记录、计算任务队列

### 4.3 表单布局

```
┌─ 表单区域 ────────────────────────────────────────────────────┐
│                                                               │
│  分组标题 <Subheading>                                          │
│  ↓ --space-3                                                  │
│  字段组（每行 1-2 个字段，gap: --space-4）                      │
│  ↓ --space-6                                                  │
│  分组标题 <Subheading>                                          │
│  ↓ --space-3                                                  │
│  字段组                                                       │
│  ↓ --space-8                                                  │
│  操作栏（主按钮左，取消右，或反之按页面惯例）                     │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

适用于：创建实验、Agent 配置、数据导入

### 4.4 详情页布局

```
┌─ 页面标题栏（标题 + 操作按钮）─────────────────────────────────┐
│  padding: --space-6                                           │
├─ 左：主内容（2/3）──────────┬─ 右：侧边信息（1/3）─────────────┤
│  <AgCard> 包装                │  <AgCard> 包装                  │
│  内容 padding: --space-6     │  元数据、标签、操作              │
│                              │                                │
└──────────────────────────────┴────────────────────────────────┘
```

适用于：化合物详情、实验详情、任务详情、报告页

## 5. 组件选择决策树

### 5.1 "该用什么容器"

```
内容需要容器吗？
│
├─ 独立信息块，需要视觉分组
│  → <AgCard>（默认 --shadow-card + --radius-md）
│
├─ 浮层/弹窗内容
│  → <AgCard variant="elevated">（--shadow-modal + --radius-lg）
│
├─ 内容只需要分割，不需要浮起
│  → border-bottom: 1px solid var(--color-border-subtle)
│
└─ 不需要容器
   → 直接排布，用间距区分
```

### 5.2 "该用什么按钮"

```
这个操作的重要性？
│
├─ 页面最重要的操作（每区块最多 1 个）
│  → <Button>（黑底白字 / 白底黑字）
│
├─ 次要操作
│  → <Button variant="secondary">（灰色描边）
│
├─ 不重要但需要可点击
│  → <Button variant="ghost">（无底无边）
│
├─ 危险操作（删除/重置/取消订阅）
│  → <Button variant="destructive">（红底白字）+ 二次确认
│
└─ 纯文字链接
   → <a> 标签，hover 用 --color-link-hover
```

### 5.3 "该用什么反馈"

```
反馈的重要程度？
│
├─ 阻断性（必须处理才能继续）
│  → <Dialog>（<AgCard variant="elevated">）
│
├─ 重要但不阻断
│  → Toast（自动消失 3-5s）
│
├─ 字段级错误
│  → 输入框下方红色文字 + 红色边框
│
└─ 成功确认
   → Toast（带绿色图标）
```

## 6. 常见翻车点与修正

| 翻车点 | 为什么翻车 | 正确做法 |
|--------|-----------|---------|
| 用品牌蓝做焦点环 | 品牌蓝限用三处 | `--color-focus-ring`（中性色） |
| 按钮用彩色填充 | AG 按钮是中性色 | Primary=黑白，只有 Danger 用红 |
| 字重用 700 | AG 上限 600 | 需要更强对比 → 改字号 |
| 自己造阴影 | 必须从 token 取 | 用 `--shadow-card` 等 |
| 大面积留白 | AG 是信息密集型 | 适度紧凑，但保持呼吸感 |
| 装饰过多 | "冷峻精确"风格 | 信息优先，装饰最少化 |
| 混用多个主图标库 | 单项目单库（白名单 MingCute / Lucide） | 风格统一；shadcn 自带 Lucide 豁免 |
| 手写 hover/focus | shadcn/AG 组件已内置 | 不要覆盖，直接用 shadcn/AG 组件 |
